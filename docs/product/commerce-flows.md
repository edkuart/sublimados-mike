# Flujos de comercio

## Flujo publico

1. Usuario visita home.
2. Explora categorias o usa busqueda.
3. Abre detalle de producto.
4. Revisa imagenes, descripcion, precio base, tiempos y politicas resumidas.
5. Selecciona variantes: talla, color, material, acabado o capacidad.
6. Completa campos de personalizacion.
7. Sube imagenes, logos o referencias si aplica.
8. Revisa resumen de cotizacion.
9. Acepta politicas necesarias.
10. El sistema guarda la cotizacion.
11. Se abre WhatsApp con mensaje prellenado.

## Flujo comprador registrado

1. Usuario inicia sesion con Google, Facebook o correo.
2. Completa perfil y datos de contacto.
3. Guarda favoritos.
4. Crea cotizaciones reutilizando datos previos.
5. Consulta historial de cotizaciones y pedidos.
6. Revisa estados.
7. Aprueba o comenta pruebas de diseno cuando aplique.
8. Contacta por WhatsApp desde una cotizacion existente.

## Flujo administrador

1. Admin inicia sesion.
2. Entra al dashboard.
3. Revisa cotizaciones nuevas y pedidos por estado.
4. Administra productos, categorias, variantes e imagenes.
5. Revisa archivos del cliente.
6. Contacta por WhatsApp si hace falta aclaracion.
7. Sube prueba de diseno.
8. Registra aprobacion del cliente.
9. Cambia pedido a produccion, listo, entregado o cancelado.
10. Consulta historial y auditoria.

## Flujo WhatsApp

Antes de abrir WhatsApp se debe crear una cotizacion persistida. El mensaje debe incluir:

- Numero de cotizacion.
- Nombre del cliente.
- Telefono.
- Producto.
- Cantidad.
- Variantes elegidas.
- Personalizacion solicitada.
- Referencias de archivos subidos.
- Precio estimado.
- Tiempo estimado de produccion.
- Tipo de entrega.
- Ciudad o zona.
- Observaciones.
- Confirmacion de aceptacion de politicas.

## Nota importante

El enlace `wa.me` no adjunta archivos automaticamente. Los archivos deben quedar guardados
en el sistema y el mensaje debe incluir el numero de cotizacion o referencias internas.
